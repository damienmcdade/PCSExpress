## 2. Verified defects

### P0 — Home "T-MINUS · REPORT-NLT" card inverted passed/upcoming  ✅ FIXED
`src/App.jsx` (TMinusDashboard). `tMinus` was computed as `-m.days - daysUntil` — the **negation** of the milestone's true distance from today (`daysUntil + m.days`). Result: `passed = tMinus < 0` marked **future** milestones as done (greyed + strike-through) and **past** ones as active, and the list sorted in reverse. Worked example (report 10 days out): "Pack-out week" (4 days ago) showed active; "Report to your new unit" (10 days away) showed struck-through as passed. This is on the home screen for every user with a report/depart date.
**Fix applied:** `tMinus: daysUntil + m.days`. Verified with a logic harness — past pack-out → passed, future report-in → upcoming.

### P2 — Move Budget Tracker showed the wrong PPM incentive rate  ✅ FIXED
`src/components/MoveBudgetTracker.jsx:27` hint read "95% of GCC", but `src/lib/ppmCalculator.js:37` uses `incentiveRate: 1.0` (100%) everywhere else (PPM estimator, Move Strategy). 95% is the pre-2021 rate.
**Fix applied:** copy now reads "100% of the Government Constructive Cost (GCC)".

## 4. Backend & security — STRONG (no exploitable findings)

Confirmed clean: **no committed secrets** (the prior `RAILWAY_TOKEN` concern exists only as `${{ secrets.RAILWAY_TOKEN }}` GitHub-Actions indirection — nothing to scrub from git; rotating the dashboard token operationally is still prudent), parameterized `pg` queries, no SSRF (fixed upstream hosts), no wildcard-CORS-with-credentials, per-route body limits, constant-time secret compare, redacted error logging, non-extractable AES-256-GCM client crypto.

Hardening backlog (low severity):
- **P1** `server/lib/pushStore.js:60` — Postgres TLS uses `rejectUnauthorized: false` on the public proxy host (no cert verification). Pin the CA / `verify-full`, matching the sibling app.
- **P2** `/api/jtr-assistant` skips the `containsLikelyPii` filter that `/api/ai` enforces — a user can send raw PII to the LLM provider. Apply the same filter or document the asymmetry.
- **P2** AI global cost cap is per-instance only and increments inside the rate-limiter (before validation). Back it with a shared store + an Anthropic account spend limit for real fleet-wide enforcement.
- **P2** `/api/push-dispatch` brute-force budget is per-IP/per-instance (10/min). Add a global failed-attempt backoff; require a high-entropy `PUSH_DISPATCH_KEY`.
